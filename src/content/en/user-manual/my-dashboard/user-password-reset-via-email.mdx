# User Password Reset via Email

### Overview

From QueryPie 10.2.2, when administrators have configured Email integration and user account authentication information exists in QueryPie's internal database, users can reset their passwords through Email.
This feature cannot be used when authentication information is external to QueryPie, such as with ID Provider or LDAP.

<Callout type="info">
**Methods for resetting user passwords in QueryPie**
* Administrators directly reset user passwords (issue temporary passwords)
* Administrators send password reset emails to users (supported from QueryPie 10.2.2)
* Users directly request password reset from the login screen (supported from QueryPie 10.2.2)
</Callout>

### Password Reset through Password Reset Email

If you have been notified by an administrator about user password reset, you should check if a password reset email has been received in your inbox.
The content of the email that users receive is similar to the following.

<figure data-layout="center" data-align="center">
<img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241218-224655.png" alt="Password Reset Email" width="610" />
<figcaption>
Password Reset Email
</figcaption>
</figure>

The password reset link provided in the email is valid for 30 minutes.
In this case, you must request password reset from the administrator again.

### Requesting Authentication Code for Direct Password Reset from Login Screen

1. Click the "Forgot your password?" link on the login page.<br/>
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241218-233331.png" alt="Forgot your password?" width="760" />
  <figcaption>
  Forgot your password?
  </figcaption>
  </figure>
2. Enter your ID in the dialog box and click the `Request` button.<br/> <br/> 
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241220-030541.png" alt="Password Reset Request" width="406" />
  <figcaption>
  Password Reset Request
  </figcaption>
  </figure>
3. Check if an authentication code for identity verification has been received in your inbox. <br/> <br/>  
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241218-234314.png" alt="Authentication Code Received via Email" width="574" />
  <figcaption>
  Authentication Code Received via Email
  </figcaption>
  </figure>
4. Enter the authentication code confirmed in the email. The received authentication code must be entered within 30 minutes. If you didn't receive the email, you can click the "Resend authentication code" link to request the authentication code email again. However, resend requests are only possible 5 minutes after the initial resend.<br/> <br/>  
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241220-030233.png" alt="Enter Authentication Code" width="319" />
  <figcaption>
  Enter Authentication Code
  </figcaption>
  </figure>
5. Enter your ID and register a new password.<br/> <br/>   <br/>  
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241219-002622.png" alt="Enter New Password" width="736" />
  <figcaption>
  Enter New Password
  </figcaption>
  </figure>
  <figure data-layout="center" data-align="center">
  <img src="/user-manual/my-dashboard/user-password-reset-via-email/image-20241219-000927.png" alt="Confirm Password Change" width="736" />
  <figcaption>
  Confirm Password Change
  </figcaption>
  </figure>
6. Log in with the changed password.  <br/>
