# Reverse Sync Test Page

QueryPie ACP製品のインストール方法は、コンテナの実行方式やインストールヘルパープログラムのバージョンに応じて、個別のガイドドキュメントが提供されます。

### Compose Toolでコンテナを実行する

一般的なインストール・実行方法であり、メーカーが推奨するインストール方法です。
Docker Composeを活用して、Container Imageをダウンロードし、実行・停止する方式です。

この場合、インストールヘルパープログラムのバージョンに応じて、2つのインストール方式が提供されます。

#### setup.v2.shでインストールする

2025年8月にリリースされたQueryPie ACP Community Editionのリリースとともに、エンドユーザーおよびパートナーエンジニアに提供されるインストールヘルパープログラムです。
Docker Hubに公開されたContainer Imageをダウンロードしてインストールする方式です。

インストール過程が自動化されており、初めてインストールするユーザーでも簡単に製品をインストールできます。
setup.v2.shが実際に実行するインストール過程はsetup.shのインストール方式と同等です。
setup.v2.shはその過程が自動化されており、Version 10など旧バージョンのインストールをサポートしていないという違いがあります。
より詳細な違いについては、このドキュメントを参照してください：[setup.shとsetup.v2.shの比較](../installation/installation/comparison-of-setupsh-and-setupv2sh)

Apple Siliconを使用するmacOS環境でのインストールをサポートしています。

詳細なインストール方法については、このドキュメントを参照してください：[インストールガイド - setup.v2.sh](../installation/installation/installation-guide-setupv2sh)

#### setup.shでインストールする

QueryPie ACP Version 9の時代からパートナーエンジニアに提供されてきたインストールヘルパープログラムです。
Harborというコンテナレジストリでアカウントを発行してもらい、Container Imageをダウンロードできます。

詳細なインストール方法については、このドキュメントを参照してください：[インストールガイド - シンプル構成](../installation/installation/installation-guide-simple-configuration)

### Kubernetes環境にインストールする

QueryPie ACPはコンテナ環境で動作するソフトウェア製品です。
Compose Toolを使用してインストール・実行するのが一般的ですが、Kubernetes環境でもスムーズに動作します。

Kubernetes環境で製品をインストールする場合、AWS EKS、GCP GKEなどのManaged Kubernetes ClusterとSelf-Hosted Clusterの両方が使用可能です。

* AWS EKS環境：[AWS EKS環境でインストールする](../installation/installation/installing-on-aws-eks)
* Helm Chart：[https://github.com/chequer-io/querypie-deployment](https://github.com/chequer-io/querypie-deployment)
