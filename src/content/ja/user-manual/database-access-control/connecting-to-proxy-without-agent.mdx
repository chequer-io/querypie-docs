# エージェントなしでプロキシ接続

### Overview

エージェントなしでURL Proxyを通じた接続もサポートします。
Redash、Tableau、ZepplinのようなウェブサービスでDBに接続しようとする時、QueryPieで生成されたURL Proxy認証情報を通じてアクセス制御政策を適用されながらも安全にデータベースにアクセスが可能です。


### URL Proxy認証情報受領

<Callout type="info">
この作業は管理者によってProxy使用が有効化されたDBコネクションに限ってのみ実行が可能です。
使用設定のために[Database Proxy使用有効化](../../querypie-overview/proxy-management/enable-database-proxy)文書を参照してください。
* 10.2.6からOracleユーザー接続ページProxy InfoにService Name/SID情報が表記されます。
</Callout>

<figure data-layout="center" data-align="center">
![QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info](/user-manual/database-access-control/connecting-to-proxy-without-agent/image-20240731-133408.png)
<figcaption>
QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info
</figcaption>
</figure>

1. QueryPieログイン後、上部メニューバーオプションで`Databases`をクリックします。
2. 右上の`Proxy Info`をクリックします。
3. 使用しようとするサービスに **Proxy Host** および **Proxy Port** をコピーして実際のDBに対するホストの代わりに入力します。
4. 下部の`Create Token`ボタンをクリックしてユーザー認証トークンを発行します。

<figure data-layout="center" data-align="center">
![QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info &gt; Create Token](/user-manual/database-access-control/connecting-to-proxy-without-agent/image-20240731-133805.png)
<figcaption>
QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info &gt; Create Token
</figcaption>
</figure>

1. 該当ポップアップで提供する **Username** および **Password** を使用しようとするサービスに入力してQueryPieプロキシを通じた接続を有効化します。（該当ユーザー認証トークン情報は最初発行時にのみ全体露出されます）
2. もしトークンキーを紛失した場合、既存キーを履歴で`Revoke`ボタンをクリックして削除し、再度`Create Token`ボタンをクリックしてトークンを新規発行して使用してください。
