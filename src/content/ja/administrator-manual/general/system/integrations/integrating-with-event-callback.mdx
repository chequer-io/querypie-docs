# Event Callback連携

### Overview

Event Callback連携はウェブフック（Webhook）を通じてQueryPieで発生するシステムイベントを顧客社のアプリケーションにリアルタイム送信する機能です。
これを通じて主要システムイベント通知を外部サービスと統合して管理できます。


### Event Callback設定

1. Administrator &gt; General &gt; System &gt; Integrationsメニューに移動します
2. Event Callbackタイルをクリックして詳細ページに移動します。
3. 詳細ページにある`+ Create`ボタンをクリックするとEvent Callbackを生成できるポップアップが表示されます。

1. Event Callback設定のために以下の情報を入力します。
    *  **Name** : 識別が容易になるよう任意の名前を入力します。
    *  **Callback URL** : イベント通知を受信する外部アプリケーションのウェブフックURLを入力します。
    *  **Status** : ウェブフックの動作状態を選択します。
        *  **Enabled** : ウェブフックが有効化されリアルタイムでイベントを送信します。
        *  **Paused** : ウェブフックを一時停止します。この状態で発生したイベントは累積され、再びEnabledに変更時累積されたイベントがすべて送信されます。
        *  **Disabled** : ウェブフックを完全に無効化します。この状態ではイベントが送信されず、累積もされません。
    *  **Subscribed Events** : 通知を購読するイベントタイプを選択します。現在サポートするイベントは以下の通りです。
        * `USER_LOCKED`: ユーザーアカウントロック
        * `USER_LOGIN_SUCCEEDED`: ユーザーログイン成功
        * `USER_CREATED`: ユーザー生成
        * `USER_DELETED`: ユーザー削除
    *  **Authorization Header** : ウェブフックリクエスト時セキュリティ認証のために必要な認証ヘッダー値を入力します。（例：Authorization: Bearer &lt;token&gt;）
    *  **Description** : 設定に対する説明を入力します。（選択事項）<br/>

1. `Create`ボタンを押して設定を保存します。

### Event Callback設定修正

1. Administrator &gt; General &gt; System &gt; Integrationsメニューに移動します。
2. Event Callbackタイルをクリックして詳細ページに移動します。
3. 生成されたEvent Callbackリストで修正したい項目をクリックします。
4. 必要な情報を修正した後保存します。

### Event Callback設定削除

1. Administrator &gt; General &gt; System &gt; Integrationsメニューに移動します。
2. Event Callbackタイルをクリックして詳細ページに移動します。
3. 生成されたEvent Callbackリストで削除したい項目の左側チェックボックスをチェックします。
4. リスト左上部で`Delete`ボタンをクリックします <br/> 
  <figure data-layout="center" data-align="center">
  <img src="/administrator-manual/general/system/integrations/integrating-with-event-callback/Screenshot-2025-08-25-at-6.14.27-PM.png" alt="Screenshot-2025-08-25-at-6.14.27-PM.png" width="736" />
  </figure>
5. 確認ウィンドウが表示されると削除を進行して設定を削除します。
