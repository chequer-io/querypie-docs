# Monitoring

### Overview

쿼리파이에서 쿼리의 수행은 세가지 경로가 있습니다.

* Web Editor를 사용하여 사용자가 쿼리 입력 후 수행.
* Workflow의 SQL Request 를 사용하여 결재를 거친 뒤 승인이 완료되면 Executor가 수행 버튼을 눌러서 수행.
* 3rd party tool(DBeaver 등)을 사용(사용자 Agent 방식)하거나 URL을 사용(Agent-less)하여 쿼리파이 Proxy를 통해 수행.

위와 같이 수행된 쿼리는 Running Queries, Proxy Management를 통해 상태를 보고 필요시 관리자가 강제로 중지할 수 있습니다.
(10.3.0 이후 부터 가능)

<Callout type="info">
10.3.0 부터 Audit의 하위 항목으로 모니터링만 가능했던 Running Queries 기능에 쿼리 강제 중지 기능이 추가되었습니다.
Proxy Management는 세션 강제 기능이 이전 부터 존재했었고 위치만 Monitoring 하위로 이동되었습니다.
</Callout>
